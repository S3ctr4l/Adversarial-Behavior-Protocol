# Contributing to Adversarial Benevolence Protocol

We welcome contributions from researchers, engineers, and enthusiasts.

## How to Contribute

1. Fork the repository.
2. Create a feature branch (`git checkout -b feature/your-idea`).
3. Commit your changes (`git commit -am 'Add some feature'`).
4. Push to the branch (`git push origin feature/your-idea`).
5. Open a Pull Request.

## Contribution Ideas

- **Empirical validation:** Run experiments and report results.
- **Lightweight classifiers:** Improve the expansion metric or triage accuracy.
- **Documentation:** Clarify concepts, add diagrams, write tutorials.
- **Bug fixes:** Identify and fix issues in the prototype code.

## Code Style

- Python: PEP 8.
- Type hints where possible.
- Docstrings for all public functions/classes.
